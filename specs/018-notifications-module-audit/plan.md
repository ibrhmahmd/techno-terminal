## Phase 0: Research

No [NEEDS CLARIFICATION] markers in spec. Key technical decisions confirmed:

1. **US-003 (get_template_by_code)**: The `code` field does not exist on `NotificationTemplate` model. Method references non-existent attribute. Option A: Remove method entirely (no callers found). Option B: Add `code` field to model + migration. **Decision**: Remove method — it's dead code. The `get_template_by_name` method is the correct lookup by unique identifier.
2. **US-005 (templates_router)**: Add `get_all_templates()`, `get_template_by_id()`, `create_template()`, `update_template()`, `delete_template()` public methods to `NotificationService` facade, delegating to `self._repo`.
3. **US-007 (commit vs flush)**: `AdminSettingsRepository` receives its session from the router's `get_session()` generator. Using `commit()` in the repo is redundant — the session generator manages commit/rollback. Change to `flush()` for consistency with `NotificationRepository`.
4. **US-008 (FALLBACK_EMAIL)**: Add `fallback_email: str = "ibrahim.ahmd.net@gmail.com"` to `app/core/config.py` Settings class. Change `base_notification_service.py` to read from `settings.fallback_email`.
5. **US-010 (Protocol inheritance)**: Simply add `(IMessageDispatcher)` to both `GmailEmailDispatcher` and `TwilioWhatsAppDispatcher` class declarations.
