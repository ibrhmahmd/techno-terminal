## Phase 0: Research

No [NEEDS CLARIFICATION] markers in spec. Research tasks are about confirming the approach for each fix:

1. **FR-001 — Enrollment date-boundary fix**: Wrap `func.coalesce(...)` with `func.date(...)` to match `DATE(COALESCE(...))` pattern used in `_fetch_revenue`. This ensures PostgreSQL casts the result to DATE, stripping time components from `created_at` timestamps.
2. **FR-002 — Status filter consistency**: Two options: (a) add `status = 'completed'` filter to `_fetch_instructor_summary`, or (b) remove it from `_fetch_session_details`. Option (a) is preferred — instructor summary should only reflect sessions the report actually shows detail for.
3. **FR-003 — exec()/execute() fix**: Change `self._repo._session.exec(stmt, params=...)` to `self._repo._session.execute(stmt, params=...)` in `base_notification_service.py:115`. Simple one-line change.
4. **FR-007 — N+1 to JOINs for payments**: Replace individual `session.get(Student)`, `session.get(Enrollment)`, `session.get(Group)` with a single JOIN query that returns all needed fields.
5. **FR-007 — N+1 to JOINs for session details**: Replace per-session `session.get(Employee)`, attendance query, and per-attendance `session.get(Student)` with JOINed queries.
6. **FR-005 — Specific exception types**: Replace `except Exception` with `except SQLAlchemyError` (or narrower) in each `_fetch_*` method. Let unexpected exceptions propagate up through `_fetch_daily_aggregates` → caller.
7. **FR-010 — Future-date validation**: Add a guard at the top of both endpoint functions checking `report_date > date.today()`, raising HTTPException 400.
8. **FR-011 — PDF name overflow**: Truncate combined student names string to ~80 characters with "..." suffix, or use smaller font for that column only.
