## Phase 0: Outline & Research Plan

### Unknowns to Research

1. **Exact test method inventory** — For each of the 15 test files, identify which specific test methods need delete/update/infra-fix, and what the precise change is.
2. **Auth bypass pattern** — Confirm the exact `override_auth` fixture pattern from `conftest.py` and how to apply it to tests currently using real Supabase JWT.
3. **422 root causes** — For tests getting 422 instead of expected status, identify the exact request body/param mismatch by running each test with `--tb=long` and cross-referencing the current endpoint schema.
4. **Notification template body** — Read the current template body from migration 059 and verify which test assertions fail against it.

### Research Dispatch

| Research Task | Method |
|---|---|
| Per-file inventory of failing tests | Read each test file, list failing methods and expected changes |
| `override_auth` fixture pattern | Read `tests/conftest.py` and `tests/utils/jwt_mocks.py` |
| 422 root cause analysis | Run each failing test with `--tb=long` and capture Pydantic validation errors |
| Notification template body | Read `db/migrations/059_update_daily_report_template_3kpi.sql` |
